## üí° Interview Talking Points

### Project Overview (30 seconds)
"I built VAM Insurance System, a full-stack web application that modernizes the insurance purchase process. The system uses AI to automatically extract information from ID cards and vehicle documents with 95% accuracy, then recommends suitable insurance packages. Users can complete the entire purchase workflow online, from document upload to payment confirmation. The tech stack includes React with TypeScript for the frontend, FastAPI for the backend, and Google Gemini AI for intelligent document processing."

### Technical Deep Dive (If asked)

**Challenge 1: Document OCR Accuracy**
"Initially tried Tesseract OCR but only achieved 60% accuracy with Vietnamese ID cards. Switched to Google Gemini 2.0 Flash with carefully engineered prompts and structured JSON output, which increased accuracy to 95% while maintaining sub-2-second response times."

**Challenge 2: Dual Form Handling**
"Encountered a bug where natural disaster insurance purchases weren't saving. Root cause was different data structures between standard forms (flat object) and natural disaster forms (nested object). Implemented TypeScript type guards and conditional extraction logic to handle both formats safely."

**Challenge 3: State Management**
"Chose Zustand over Redux because the application needed simple, performant state management without boilerplate. This reduced state management code by 90% and improved development velocity significantly."

- **Performance**: First load < 1.5s, API responses < 2s
